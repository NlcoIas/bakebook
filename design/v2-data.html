What's different, in code

Three new screens, plus changes to the existing journal and home (small stats strips). The schema needs nine new fields and one bundled JSON file (per-100 g nutrition for ~80 baking ingredients, derived once from USDA FoodData Central, shipped with the app — no external API call at runtime).

Schema deltas

-- recipes: add a real number for serving math
ALTER TABLE recipes ADD COLUMN servings INT DEFAULT 1;
ALTER TABLE recipes ADD COLUMN serving_g NUMERIC(7,2);   -- weight per slice/portion

-- pantry items hold default cost + nutrition refs
CREATE TABLE pantry_items (
  id              UUID PRIMARY KEY,
  name            TEXT UNIQUE NOT NULL,         -- 'Zopfmehl', 'Bread flour'
  cost_per_kg     NUMERIC(7,2),                  -- CHF
  cost_currency   TEXT DEFAULT 'CHF',
  nutrition_ref   TEXT,                          -- key into bundled JSON
  default_role    TEXT,
  created_at      TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE pantry_price_history (
  id              UUID PRIMARY KEY,
  pantry_item_id  UUID REFERENCES pantry_items(id),
  cost_per_kg     NUMERIC(7,2) NOT NULL,
  observed_at     TIMESTAMPTZ DEFAULT now(),
  source          TEXT                            -- 'manual' | 'migros' | 'coop'
);

-- recipe_ingredients can override the pantry default
ALTER TABLE recipe_ingredients ADD COLUMN pantry_item_id UUID REFERENCES pantry_items(id);
ALTER TABLE recipe_ingredients ADD COLUMN cost_override_per_kg NUMERIC(7,2);
ALTER TABLE recipe_ingredients ADD COLUMN nutrition_override JSONB;
                  -- { kcal, protein, fat, carbs, sugar, fiber, salt }, all per 100 g

-- bakes: structured outcomes
ALTER TABLE bakes ADD COLUMN kitchen_temp_c    NUMERIC(4,1);
ALTER TABLE bakes ADD COLUMN kitchen_humidity  NUMERIC(4,1);
ALTER TABLE bakes ADD COLUMN final_weight_g    NUMERIC(7,2);
ALTER TABLE bakes ADD COLUMN start_weight_g    NUMERIC(7,2);
ALTER TABLE bakes ADD COLUMN rise_height_cm    NUMERIC(4,1);
ALTER TABLE bakes ADD COLUMN internal_temp_c   NUMERIC(4,1);
ALTER TABLE bakes ADD COLUMN crumb_score       INT CHECK (crumb_score BETWEEN 1 AND 5);
ALTER TABLE bakes ADD COLUMN crust_score       INT CHECK (crust_score BETWEEN 1 AND 5);
                  -- water_loss_pct and oven_spring_pct are computed views, not columns

-- recipes: ratios are all computed; only one new column needed
ALTER TABLE recipes ADD COLUMN target_dough_g  NUMERIC(7,2);   -- nullable; for "scale to" mode
Tier 1 · ship in M3

Confirmed for v1

  • Macros (per serving + per 100 g) with bundled USDA-derived JSON
  • Cost per recipe + per serving, with default cost_per_kg on pantry items
  • Full baker's ratios — hydration, fat, sugar, salt, leaven, prefermented flour
  • Allergen badges (auto-derived from ingredients with manual override)
  • Ready-by reverse calculator
  • Environmental context capture at bake start (kitchen temp, optional humidity)
  • Structured outcome capture on the reflection screen
Tier 2 · ship in M4 or M5

The Insights screen

  • Bakes per month bar chart, year-on-year
  • Top tweaks ranked across all bakes
  • Equipment success rates
  • Calendar heatmap, GitHub-style
  • Stats strip: total bakes, avg rating, flour used, money spent
  • "Pattern" callouts written by hand: surface signals from the tweaks data
